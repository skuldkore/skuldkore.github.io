Agents can act faster than teams can control.

AI agents are becoming operational actors. They can inspect private repositories, write patches, run commands, create artifacts, spawn subtasks, and continue across long-running sessions. Without a control plane, autonomy turns into drift.

Unclear authority

Who approved the action? Which identity requested it? Which policy allowed it? Most agent stacks cannot answer cleanly.

Weak audit trail

Chat logs are not execution records. Teams need structured state, decisions, artifacts, approvals, and replayable action history.

Unsafe side effects

Reads, writes, tool calls, network access, retries, and spawned agents need policy gates before they hit real code or infrastructure.

Governed swarm execution for real agent work.

Skuld separates behavior from execution. Agents and profiles propose actions. The supervisor validates, schedules, gates, records, and executes them through controlled paths.

Governed swarm execution

Coordinate multiple agents while keeping side effects behind supervisor policy, sandboxing, capability boundaries, and approval gates.

Typed state

Represent findings, decisions, checkpoints, tasks, artifacts, attempts, events, and approvals as structured operational state.

Operator approval

Require approval for code mutation, command execution, external access, agent spawning, or high-impact workflow changes.

MCP integration

Expose ergonomic agent and tool interfaces without making MCP the trust boundary. Enforcement remains inside the control layer.

Audit log and replay

Track what happened, why it happened, what policy allowed it, and how the system can recover or replay after failure.

Profiles package the mission, not just the prompt.

A Skuld profile is not a prompt preset. It packages behavior, UI, commands, agent roles, workers, schemas, and runtime feeds into a mission-specific operating mode.

A profile defines what agents exist, what commands the operator can issue, what state is written, what workers feed the runtime, and what the TUI shows. The profile is the operating mode for the swarm.

A control plane, not another agent wrapper.

Skuld keeps the kernel domain-agnostic. Profiles define behavior and UI. The supervisor enforces policy. The runtime executes through observable, replayable action paths.

01

User / CLI / TUI

Operator commands, approvals, views, and status.

02

Profile package

Behavior hooks, prompts, commands, workers, schemas, strategy, and "ui/layout.yaml".

03

Supervisor

Policy, approval, sandbox validation, scheduling, and action enforcement.

04

Swarm runtime

Agents, workers, IPC, outputs, artifacts, and task execution.

05

Audit / replay

Execution ledger, typed events, recovery, and traceability.
